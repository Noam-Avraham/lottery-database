# Israel Lotto Archive — Database, Statistics & Website

## Overview

This project scrapes all available lottery results from [pais.co.il](https://www.pais.co.il/lotto/archive.aspx), stores them in a local SQLite database, generates statistical graphs, and serves a Flask website for exploring the data.

---

## Quick start

```bash
# Install dependencies (one time)
pip install -r requirements.txt
pip install flask

# Start the website
python app.py
```

Open http://127.0.0.1:5000 in your browser.

## Database — `lottery.db`

### What's in it

The database contains **1,672 draws** from the **modern lottery format only**, covering:

- **Date range:** March 5, 2011 → present
- **Main number pool:** 1–37
- **Strong (bonus) number pool:** 1–7

### Why this date range

The Israeli Lotto changed its format several times:

| Era | Dates | Main pool | Strong pool |
|-----|-------|-----------|-------------|
| Old (excluded) | 1999–Mar 2004 | 1–49 | — |
| Middle (excluded) | Mar 2004–Feb 2009 | 1–34 | 1–10 |
| Transitional (excluded) | Feb 2009–Mar 2011 | 1–37 | 1–8 |
| **Modern (kept)** | **Mar 2011 → now** | **1–37** | **1–7** |

Only the modern format is kept in `lottery.db` so all draws are directly comparable.
Older formats are preserved in `with old result/`.

### Tables

#### `lotteries`
One row per draw.

| Column | Type | Description |
|--------|------|-------------|
| `id` | INTEGER | Draw ID (primary key) |
| `draw_date` | TEXT | Date in DD/MM/YYYY format |
| `draw_time` | TEXT | Time of draw (HH:MM) |
| `num1`–`num6` | INTEGER | The 6 winning numbers (1–37, sorted ascending) |
| `strong_number` | INTEGER | The strong (bonus) number (1–7) |
| `first_prize_lotto` | INTEGER | Planned first prize — regular lotto (NIS) |
| `first_prize_double` | INTEGER | Planned first prize — double lotto (NIS) |
| `total_prizes` | INTEGER | Total prize pool distributed (NIS) |
| `scraped_at` | TEXT | Timestamp when the row was scraped |

#### `prize_tiers`
Prize distribution per draw — one row per tier per game type.

| Column | Type | Description |
|--------|------|-------------|
| `id` | INTEGER | Auto-increment primary key |
| `lottery_id` | INTEGER | Foreign key → `lotteries.id` |
| `game_type` | TEXT | `regular` or `double` |
| `tier` | TEXT | Match tier: `6 + חזק`, `6`, `5 + חזק`, `5`, `4 + חזק`, `4`, `3 + חזק`, `3` |
| `winners` | INTEGER | Number of winners at this tier |
| `prize` | INTEGER | Prize per winner in NIS |

#### `skipped_ids`
IDs that returned no data (pre-modern draws, gaps in numbering).

## Statistical graphs

Two PNG files are generated by `stats.py` and served by the website.

### `stats_main_numbers.png` — The 6 Main Numbers

Three charts stacked vertically:

1. **Total appearances (bar chart)** — how many times each number (1–37) appeared across all draws. Average line shown; top-3 and bottom-3 annotated.
2. **Appearances by day of week (heatmap)** — percentage of draws on each day that included each number. Normalised per day so Tuesday, Saturday, and Thursday are directly comparable.
3. **Appearances by jackpot prize tier (heatmap)** — how often each number appeared grouped by the planned first-prize amount. Prize buckets: `<5M`, `5-15M`, `15-25M`, `25-35M`, `35M+` (NIS).

### `stats_strong_number.png` — The Strong Number

Same three charts for the strong number (1–7). Should be approximately uniform (~14.3% per number per draw day).

## Website pages

### Home (`/`)
Overview of the project: database stats, feature summary, format history timeline, database schema.

### Search (`/search`)
Two search modes:

- **By date** — enter a day and month (no year) to find all draws on that date across all years, with number frequency stats.
- **Combination checker** — enter up to 6 main numbers (1–37) and optionally a strong number (1–7) to find all draws where those numbers appeared, with the prize tier and amount you would have won.

### Interesting Facts (`/facts`)
Automatically computed statistics:
- Overview cards: total draws, coverage %, repeated combinations, average sum, jackpot count
- Hot & cold numbers (last 50 draws)
- Most common pairs and triplets
- Jackpot records: biggest prize, most winners in one draw, longest drought, most common prize size
- Frequency graphs (click to zoom)

## Scripts

| Script | Purpose |
|--------|---------|
| `app.py` | Flask web server — `python app.py` to start |
| `update.py` | Add/refresh individual draws and regenerate graphs |
| `stats.py` | Generate the two PNG frequency graphs |
| `scraper.py` | Full archive scraper (run once, or to back-fill) |

### update.py usage

```bash
python update.py --latest          # scrape the next draw after the current max ID
python update.py 3910              # scrape a specific draw
python update.py 3910 3911 3912    # scrape multiple draws
python update.py --latest --no-stats  # update without regenerating graphs
```

See `MAINTENANCE.md` for the full maintenance workflow.
